## 最得意/或有价值/或有苦劳的工作清单
---
* **最得意:** 分析需求和设计数据库。结合到店点餐的具体情境分析，去除掉冗余的数据库存储功能，精简数据库设计，是数据库的资源消耗极大地减少。
* **最有价值:** 将数据库模型中对应数据表的操作分别封装成一个类，实现面向对象，使得逻辑层对表进行操作时，无需考虑 sql 语句，只需调用类中所提供的增删改查操作，即可完成对于数据库的操作。
* **最有苦劳:** 功能模型图的绘制非常费时费力。
